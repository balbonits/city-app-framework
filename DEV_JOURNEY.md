# DEV_JOURNEY

## Purpose
Developer journal capturing discussions, thought processes, and decisions made with AI for the City App Framework.

## Entries
- **2025-08-23**: Initialized framework v0. Defined city-themed structure (City Hall, Works, Components, Arts, Store, Models, Districts, Specs). Focused on modular, hook-centric design, mobile-first UX, minimal dependencies.
  - Decisions: Prioritized TypeScript, ESLint (Airbnb), no Prettier. Integrated feature flagging with variants via URL/env. Added security utilities in Works.
  - Discussions: Clarified City Hall as FE state hub, Works for hooks/utilities. Expanded to BE scaffolding (Express APIs, routing).
  - Challenges: Adapted framework for CSR/SSR/PWA/Native, balanced FE/BE without back-end experience.

- **Future Entries**: Log AI interactions, decisions, refinements (e.g., new utilities, optimizations).
